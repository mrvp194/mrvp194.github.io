Arrays vs. Hashes

10/26/2014

Ruby's arrays and hashes are fairly similar but differ in one distinct way. An array is an integer-indexed collection of any object. This is what it looks like:

array_name = [2, "dog", 6.23]
When I say integer-indexed I mean that each object within the array is assigned an integer starting with 0. so if we wanted to print the second object of the above array it would look like:
array_name[1]

"dog"
A hash on the other hand is a collection of key_value pairs. It's very similar to an array except instead of each object pointing to an integer, they now point to the key which we assign it. This looks like:
pet_names = {
     "cat" => "Zelda",
     "dog" => "Dash",
     "snake" => "Snape"
}
So the keys in the example above are "cat", "dog", and "snake". And the values are "Zelda", "Dash", and "Snape". If we wanted to print one of the pet names it would look like this:
pet_names["cat"]

"Zelda"
A hash by default returns nil when indexed by a key it doesn't contain. This can be changed like so:
name = Hash.new(20)
20 is now the new default value.
