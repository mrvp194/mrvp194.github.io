Test-Driven Development

11/30/2014

Test-driven development is a commonly used software development process that relies on a developer initialy writing test code that defines a new feature or improvement to the program. The developer then produces the minimum amount of code necesary to past the tests and finishes by refactoring that code.

There are various up sides to TDD. One may be that it helps ensure that the application is written for testability. Other reasons to use TDD are writing the tests first drives a deeper and earlier understanding of the product requirements, ensures the effectiveness of the test code, and maintains a continual focus on the quality of the product.
