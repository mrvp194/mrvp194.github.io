Ruby Classes

11/9/2014

Today I'm going to discuss classes in ruby and when to use instance vaiables and methods. I'll start off with an explanation of local variables. A Local Variable is a variable that is defined in a method. Local variables cannot be accessed outsite of the method. Local variable begin with a lowercase letter or _ and are most likely the kind of variables you've been using up to this point.

To be able to access a variable(within a class) outside of the method it was defined in, you will have to create an Instance Variable. Instance Variables start with @ followed by the variable name. This will come in handy when creating a class that has multiple methods. For example:

class Findname

def initialize(name)
@name = name
end

def hello
puts "Hello #{@name}! How are you doing?"
end
end

name = Findname.new(Michael)
name.hello
=> "Hello Michael! How are you doing?"
In the example above @name is the instance variable. Take note at how we define it in the first method and then call it in the second method. You could not do this with a local variable because it is only stored within the method it is defined in.

Instance methods are fairly simple to their variable counterparts in that they allow you to call them anywhere within the class they were defined. Initialize and hello in the above example are both instance methods.
