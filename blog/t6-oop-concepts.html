Ruby Blocks, Procs, and Lambdas

11/17/2014

As the title suggests, today I am going to discuss the differences between blocks, procs and lambdas in ruby. Referred to as closures in Computer Science, they are a vital part of ruby. To start off, blocks are the most common way to use a closure in Ruby. They look like this:

array = [3, 2, 9, 4, 2]
array.map! {|num| num + 5} => [8, 7, 14, 9, 7]

In the above example {|num| num + 5} is the block of code being run. Procs, short for procedures, are very similar to blocks. Procs have the handy ability to be saved and used multiple times while keeping your code D.R.Y. Here is an example where this might be used:

array_1 =[23, 3, 6, 9, 3]
array_2 = [4, 53, 9, 2, 5]
array_3 = [12, 4] double = Proc.new do |num|
num * 2
end
array_1.map!(&double)
array_2.map!(&double)
array_3.map!(&double)

=> [46, 6, 12, 18, 6]
=> [8, 106, 18, 4, 10]
=> [24, 8]

In the above example take note of how we created the variable double and set it equal to a new prc that runs a block of code which multiplies every element given by 2. We then refer to it later with &double to show that it is a block of code that we want run.

Lastly, lambdas differ from procs in two ways. First, unlike procs, lambdas will check the number of arguments passed like so:

def sentence(code)
one = 1
two = 2
code.call(one, two)
end


sentence(Proc.new{|x,y,z| puts "I need #{x} spoons, #{y} forks, and #{z.class} knives})

=> I need 1 spoons, 2 forks, and NilClass knives

sentence(lambda{|x,y,z| puts "I need #{x} spoons, #{y} forks, and #{z.class} knives})

=> ArgumentError: wrong number of arguments (2 for 3) (ArgumentError)

In the method sentence we use .call to make a method that can run a block of code. As you can see lambda returns an error. The other difference between lambdas and procs is lambdas will return their value to the method without ending the method. For example:

def proc
Proc.new { return "Proc.new"}.call
return "finished proc method"
end

def lambda
lambda { return "lambda"}.call
return "finished lambda method"
end

p proc
p lambda

=> Proc.new
=> finished lambda method

As you can see while the proc method returns the first block of code it runs, lambda returns the second without printing "lambda".
