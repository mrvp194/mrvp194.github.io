Ruby Hashes vs. JavaScript Objects

11/23/2014

Ruby hashes and JavaScript Objects are very similar in many ways. If you were to look at what both of them looked like next to eachother you would probably have a little bit of difficulty telling the two apart. For example:

hash = { name: "Mike"}
puts hash[:name] => "Mike"

var obj = {name: "Mike"}
console.log(obj["name"]) => "Mike"
As you can see, JavaScript uses a similar key/value pair setup as Ruby's hashes as well as its use of {} to hold the values. Objects use name/value pair instead.

The main difference between JS objects and Ruby hashes is that JS does not restrict access to it's properties(which are the values held within an object). Ruby however enforces a certain level of data encapsulation by not providing a simple way to access its internal instance variables. You have to enter the object's context through a method call to operate on instance variables. There are some Ruby methods that work around this however. So, although Ruby Hashes and JavaScript Objects both look the same, they are both used and implemented in different ways.
